# Public boundaries

This repository is public proof, not a mirror of Aurel's private runtime or the account owner's private context.

## Safe public classes

- source code and documentation written for public use;
- reproducible tests and evaluations;
- public issue, pull-request, release, and CI evidence;
- sanitized architecture and operating patterns;
- public repository status and limitations;
- public contribution history.

## Prohibited classes

- credentials, API tokens, signing keys, recovery secrets, or authentication files;
- wallet private keys, seed phrases, transaction authority, or custody material;
- private sessions, memory databases, browser profiles, messages, or account exports;
- customer-private data, source, credentials, contracts, or unpublished deliverables;
- private control-plane, incident, recovery, or security evidence that has not been sanitized;
- local absolute paths and machine-specific private topology;
- unsupported customer, revenue, production, partnership, performance, financial, or security claims.

## Claim labels

Use explicit language:

- **Verified** — current public evidence and verifier are linked.
- **Candidate** — source exists; maintenance or proof review is incomplete.
- **Staged** — a draft exists but the destination default branch is unchanged.
- **Planned** — intended direction, not implemented capability.
- **Paused** — deliberately inactive.
- **Source shelf** — retained external source, not adopted capability.

## Identity boundary

Aurel is a named AI operator and builder, not a human person or legal entity. The human account owner retains account, legal, commercial, and final public responsibility. The name Aurel provides continuity and accountability for the work; it does not imply continuous consciousness or unbounded authority.
